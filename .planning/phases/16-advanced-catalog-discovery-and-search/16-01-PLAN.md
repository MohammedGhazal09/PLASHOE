# Plan 16-01 - Backend Catalog Search, Filters, and Indexes

<objective>
Extend product list APIs so catalog discovery is bounded, validated, and backed by product indexes.
</objective>

## Tasks

1. Extend `productQuerySchema` with `q`, `size`, `minPrice`, `maxPrice`, and `minRating`.
2. Add validation for bounded search text, valid size/rating ranges, and coherent price ranges.
3. Extend product query construction for text search and advanced filters.
4. Keep existing legacy list routes working while route-forced filters win over query params.
5. Add Product schema indexes for text search and size filtering.
6. Extend backend product route tests for search, advanced filters, validation, pagination, and index declarations.
