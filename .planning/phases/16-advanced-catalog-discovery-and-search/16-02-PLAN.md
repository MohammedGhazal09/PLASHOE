# Plan 16-02 - Frontend URL-State Catalog Discovery

<objective>
Make catalog discovery controls visible and persist search/filter/sort/page state through shareable URLs.
</objective>

## Tasks

1. Add a `useCatalogUrlQuery` hook that restores query-string state and writes search/filter/sort/page changes back to the URL.
2. Wire `/collection`, `/men`, `/women`, and `/sale` catalog pages through the URL-state hook.
3. Ensure route-forced filters are applied to API params but omitted from shareable URLs.
4. Expand ProductGrid controls for search, category, sort, size, price range, minimum rating, and sale-only filtering.
5. Add visible loading, fallback/error, and no-results text states.
6. Extend catalog service fallback/defensive filtering to match the backend discovery params.
7. Add frontend tests for the hook, API wrapper, catalog service, and ProductGrid behavior.
