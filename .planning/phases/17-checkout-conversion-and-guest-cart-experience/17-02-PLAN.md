# Plan 17-02 - Frontend Checkout Intent, Cart Merge, and Saved Address UX

<objective>
Wire the account-required checkout policy into the shopper flow so local cart contents merge after auth, conflicts are reviewable, and saved addresses reduce repeated entry.
</objective>

## Tasks

1. Add `cartApi.mergeItems`.
2. Add `isBackendCartProductId`, `hasLocalCartItems`, and `mergeLocalCart` to `cartStore`.
3. Add safe user serialization for auth responses and add `authStore.addAddress`.
4. Update Account login/register to merge local cart items when returning to checkout and route conflicts to Cart.
5. Add cart/checkout review alerts for unresolved local items.
6. Update Checkout to use default saved address, save address when requested, and block payment while local items remain.
7. Add/update frontend tests for cart store merge, Account checkout return, Checkout blocked state, saved address prefill/save, and ProtectedRoute redirect state.
