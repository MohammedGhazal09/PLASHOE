# Plan 17-01 - Backend Cart Merge Contract

<objective>
Add a protected cart merge API that safely reconciles guest cart payloads into the authenticated backend cart without duplicates or partial backend mutations.
</objective>

## Tasks

1. Extend `Backend/validators/cart.js` with a strict merge payload schema for `items`.
2. Add `mergeCartItems` to `Backend/controllers/cartController.js`.
3. Aggregate incoming duplicate product/size lines before mutation.
4. Validate all affected products exist and final quantities do not exceed current stock.
5. Preserve existing backend cart coupon state and existing line prices; use current product price for newly merged lines.
6. Mount `POST /api/cart/merge` in `Backend/routes/cartRoutes.js`.
7. Add `Backend/test/cart.test.js` coverage for auth rejection, duplicate merge, stock conflict no-op, and missing product conflict.
