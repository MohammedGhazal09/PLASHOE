# Phase 02 Plan 01: Backend Test Runner and Importable App Harness

<objective>
Create the backend test foundation: a one-shot Vitest command, importable Express app, disposable MongoDB setup, shared factories, and a smoke test proving route tests can run without production MongoDB or binding `PORT`.
</objective>

## Recommendation

Keep the app extraction mechanical. Move the existing Express app construction into `Backend/app.js`, then make `Backend/server.js` import `app`, call `dotenv.config()`, call `connectDB()`, and listen on `PORT`. Do not add a `src` folder or change route/controller behavior in this plan.
