# Phase 06 Plan 03: Admin List Pagination, Frontend Wrappers, and API Docs

<objective>
Apply the shared admin pagination/filtering contract to coupon and contact admin lists, add frontend admin API wrappers for the new backend contracts, and update API documentation so execution and verification have one clear source of truth.
</objective>

## Recommendation

Finish the phase by applying the shared list contract to existing admin coupon/contact endpoints, then add wrapper methods and docs after the backend route names and fulfillment contract are stable.

## Artifacts This Plan Produces

- Extended coupon/contact query validators for admin list filters.
- Updated coupon/contact admin list controllers returning the shared pagination envelope.
- Coupon and contact schema indexes for approved filters.
- `Backend/test/admin-list.test.js` for coupon/contact admin list auth, pagination, and filters.
- `Frontend/Ecommerce-main/my-app/src/api/adminApi.js` and `adminApi.test.js`.
- `docs/API.md` updates for Phase 06 endpoints, envelopes, transition rules, errors, and wrapper mappings.

<threat_model>
Threats addressed: unbounded admin coupon/contact reads, unsafe text filters, inconsistent list envelopes across admin resources, undocumented fulfillment/error contracts, and frontend code calling ad hoc Axios paths instead of a shared wrapper. This plan does not add admin pages, move coupon/contact routes, or split existing public contact/coupon wrappers.
</threat_model>
