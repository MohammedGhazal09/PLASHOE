# Phase 01 Plan 02: Contract Checker and Evidence Alignment

<objective>
Align the focused spike checker and generated evidence with the Phase 1 source fixes. The checker should report zero Phase 1 `FAIL` findings after Plan 01-01, while preserving payment and inventory `WARN` findings as known later-phase work.
</objective>

## Context

Read these before editing:

- @.planning/phases/01-core-flow-stabilization/01-SPEC.md
- @.planning/phases/01-core-flow-stabilization/01-CONTEXT.md
- @.planning/spikes/001-core-flow-contract-check/check-contracts.mjs
- @.planning/spikes/001-core-flow-contract-check/results.json
- @.planning/spikes/001-core-flow-contract-check/contract-report.md

## Recommendation

Treat the checker as a narrow acceptance gate, not a full test suite. Update its patterns only when they no longer recognize the correct implementation or when they miss a Phase 1 acceptance detail that the plan now depends on.

## Artifacts This Phase Produces

- A still-runnable `.planning/spikes/001-core-flow-contract-check/check-contracts.mjs`.
- Updated `.planning/spikes/001-core-flow-contract-check/results.json` only if counts semantically move from failing to passing.
- Updated `.planning/spikes/001-core-flow-contract-check/contract-report.md` only if it records the same semantic transition.
