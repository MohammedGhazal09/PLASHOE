<objective>
Convert the frontend test suite to Vitest and remove recurring frontend test/build warning noise without changing storefront behavior or test intent.
</objective>

## Context

- Plan 10-01 creates the Vite/Vitest package and config foundation.
- Current CRA tests pass 18 suites and 65 tests.
- Current tests use `jest.*` globally across API, store, page, component, hook, config, and app-shell tests.
- `ProtectedRoute.test.jsx` already shows the router future-flag pattern; `CheckoutReturn.test.jsx` lacks it.
- `OrderDetail.jsx` currently triggers the exact build warning Phase 10 must remove.

## Recommendation

Perform a broad but mechanical test API conversion first, then clean the known warning surfaces. Keep tests behavior-focused and use helper extraction only where it reduces duplicate router setup or expected console handling.

## Artifacts This Plan Produces

- `Frontend/Ecommerce-main/my-app/src/test/routerTestUtils.jsx`
- Vitest-compatible updates across existing `src/**/*.test.*` files.
- Updated `Frontend/Ecommerce-main/my-app/src/setupTests.js`
- Refactored `Frontend/Ecommerce-main/my-app/src/pages/OrderDetail.jsx`

<threat_model>

### T-10-04: Fake Migration Through Jest Shim

Risk: adding a global `jest` alias would hide migration gaps and make future Vitest failures harder to debug.

Mitigation: convert mocks/spies/timers to `vi.*` and verify `rg -n "jest\\." src` returns no source test matches.

### T-10-05: Warnings Hidden Globally

Risk: globally suppressing console warnings or errors can hide real regressions.

Mitigation: use scoped spies in only the tests that intentionally trigger expected error paths, and restore each spy in the same test or hook.

### T-10-06: Hook Warning Fixed by Suppression

Risk: disabling the React hook lint warning leaves stale closure behavior in order loading.

Mitigation: refactor `OrderDetail.jsx` with `useCallback` or effect-local async logic and verify the production build is warning-clean.

</threat_model>
