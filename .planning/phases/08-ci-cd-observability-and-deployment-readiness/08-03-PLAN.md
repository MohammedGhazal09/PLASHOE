<objective>
Finish Phase 08 by correcting environment templates, documenting deployment readiness, and recording exact verification evidence across CI, backend readiness/logging, docs, and templates.
</objective>

## Context

- This plan depends on Plan 08-01 and Plan 08-02 because docs and verification must describe actual CI/readiness/logging behavior.
- Existing configuration docs say env templates are mismatched.
- Live deployment, production credentials, host selection, Docker, external APM, and platform-specific rollback commands are out of scope.
- The final verification artifact is the phase proof for all OPS requirements.

## Recommendation

Use Plan 08-03 as the closeout and verification plan. Keep docs operational and concrete, but platform-neutral. Do not invent production URLs, credentials, or host-specific commands.

<threat_model>

### T-08-07: Secret-Looking Examples Enter Source Control

Risk: env templates or docs include real or realistic credentials.

Mitigation: use obvious placeholders and static checks for secret-looking values.

### T-08-08: Deployment Runbook Assumes a Host

Risk: docs prescribe Render/Vercel/Netlify/Railway-specific commands without a selected platform.

Mitigation: keep docs platform-neutral and provide generic managed-backend/static-frontend guidance.

### T-08-09: Phase Closes Without Evidence

Risk: planning and docs appear complete but source/test/audit/readiness behavior is unproven.

Mitigation: run the full command suite and record exact results in `08-VERIFICATION.md`.

</threat_model>

Add the deployment readiness runbook:
