<objective>
Build the admin order management workflow: list/filter orders, inspect order detail, and submit fulfillment updates through the existing admin order API.
</objective>

## Context

- Phase 6 backend admin order APIs are already implemented and tested.
- `adminApi.js` already has order list/detail/fulfillment methods.
- This plan should not change backend fulfillment semantics.

## Recommendation

Implement order management as the first real admin screen because it exercises the shell, table, filters, detail, form, error, and submit patterns that later admin resources can reuse.
