<objective>
Expose product, coupon, and contact-message admin operations from frontend screens using existing protected backend endpoints.
</objective>

## Context

- Product, coupon, and contact backend admin endpoints already exist.
- Frontend `adminApi.js` lacks wrappers for product mutations, coupon create/delete, and contact mark-read/delete.
- Product/coupon/contact UI must not send unknown fields because backend validators use strict schemas.

## Recommendation

Extend the admin integration layer first, then implement resource screens that use only existing backend-supported operations. Keep upload, bulk import, analytics, and role-management work deferred.
