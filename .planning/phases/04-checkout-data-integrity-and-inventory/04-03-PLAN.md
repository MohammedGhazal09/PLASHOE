## Recommendation

Make `cartStore.js` the single cart shape boundary. Components should render normalized fields and pass normalized mutation IDs. Keep UI layout unchanged; this is a data-contract and conflict-handling task, not a visual redesign.

## Artifacts This Plan Produces

- Pure normalized cart item mapper and persisted-store migration in `cartStore.js`.
- Updated cart selectors using `unitPrice` and `lineTotal`.
- Updated checkout, cart page, and cart sidebar components without page-level mixed-shape fallback helpers.
- `ordersApi.create` support for `Idempotency-Key` and checkout generation of stable attempt keys.
- Frontend tests for backend sync normalization, guest persisted-shape migration, idempotency header propagation, and checkout conflict handling.
- API and testing docs for checkout idempotency, `409` conflict payloads, stock/coupon behavior, collision-safe order numbers, and Phase 04 commands.

<threat_model>
Threats addressed: UI clearing carts after backend checkout conflict, retrying checkout without an idempotency key, stale local cart shapes producing wrong totals or mutation IDs, duplicated item-shape fallback logic drifting across screens, and undocumented API behavior that frontend callers cannot implement correctly.
</threat_model>
