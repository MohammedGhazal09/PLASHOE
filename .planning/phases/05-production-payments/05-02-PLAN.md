# Phase 05 Plan 02: Checkout-Start Payment Session Integration

<objective>
Turn `POST /api/orders` into authenticated payment-start: create or reuse one local pending order, create a hosted Stripe Checkout Session from server totals, return the checkout URL, and compensate local side effects if provider session creation fails.
</objective>

## Recommendation

Keep local checkout and Stripe session creation as two explicit stages. The first stage owns stock/coupon/cart/order consistency. The second stage owns provider creation and must either attach provider data or run compensation that leaves no usable unpaid checkout artifact.
