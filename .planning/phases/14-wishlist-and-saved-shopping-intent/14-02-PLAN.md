<objective>
Create frontend wishlist integration primitives: API wrapper, persisted store, backend-safe id handling, guest local behavior, authenticated sync, and merge support.
</objective>

## Context

- No frontend wishlist API wrapper or store exists.
- Cart store already demonstrates guest/local and authenticated/backend persistence.
- Product normalization can produce backend ids or local fallback ids.

## Recommendation

Build state primitives before UI. The UI should call a small stable store API rather than duplicate guest/auth/local-only decisions in components.
