# Phase 20 UI Spec

## Product Detail
- If a backend product is out of stock, show a back-in-stock request form.
- Require email and explicit opt-in checkbox before submitting.
- Show a concise success or error state without implying a provider message was already sent.
- Render recommendation reason copy under recommended product cards when available.

## Order Detail
- Add a buy-again action in the order sidebar.
- On success, open the cart and show how many items were restored.
- If every item is unavailable, show an error and leave the cart untouched.

## Recovery Boundary
- Document cart recovery as opt-in intent and provider-deferred, not an active messaging workflow in this phase.
