# Plan 22-02: Address Book Default Management

## Steps
1. Add a protected backend route for setting an existing address as default.
2. Preserve one practical default when deleting a default address.
3. Add frontend API and store actions for delete address and set default address.
4. Add address form and saved-address list controls to the settings tab.

## Verification
- Backend auth tests cover set-default, default promotion after delete, validation, and auth protection.
- Store tests cover address list replacement after add, delete, and set-default calls.
- Account page tests cover add-address and set-default interactions.
