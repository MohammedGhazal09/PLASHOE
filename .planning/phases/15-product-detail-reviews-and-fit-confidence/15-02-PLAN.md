# Plan 15-02 - Verified Purchase Review API and Aggregation

<objective>
Add persisted verified-purchase reviews and product rating/fit aggregation.
</objective>

## Tasks

1. Add `Review` model with product/user refs, rating, title, comment, fit enum, verifiedPurchase, isApproved, and unique product/user index.
2. Add strict review validators for list query and create body.
3. Add controller functions for public listing and protected create.
4. Implement verified-purchase check against existing orders without changing checkout rules.
5. Implement aggregation helper to update Product rating, reviewCount, ratingDistribution, and fitSummary after approved review creation.
6. Mount review routes under `productRoutes.js` before `/:id`.
7. Add factories and backend tests for auth, verified-purchase, validation, duplicate, public listing, aggregate update, and limited user data.
