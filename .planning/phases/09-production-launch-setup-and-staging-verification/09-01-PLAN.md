<objective>
Create the Phase 09 setup evidence surface so staging topology, external configuration, database isolation, public frontend config, MapTiler status, and known blockers are explicit before hosted smoke checks run.
</objective>

## Context

- Phase 08 produced deployment docs and an incomplete user setup checklist, but no real staging URLs or hosted setup proof.
- Phase 09 requires exact evidence artifacts but forbids real secrets, raw dashboard payloads, and secret-bearing screenshots.
- Create React App embeds `REACT_APP_*` values at build time, so the staging frontend API URL must be configured before the static build.
- MapTiler browser keys are public and should be domain-restricted when enabled.

## Recommendation

Start with evidence scaffolding and redacted setup checklists. This lets later plans fill concrete hosted evidence without inventing values or mixing dashboard-only work into source code.

## Artifacts This Plan Produces

- `.planning/phases/09-production-launch-setup-and-staging-verification/09-USER-SETUP.md`
- `.planning/phases/09-production-launch-setup-and-staging-verification/09-VERIFICATION.md`
- Updated placeholder/example wording in docs/templates only if current wording can be mistaken for production-ready values

<threat_model>

### T-09-01: Secret Leakage Through Evidence

Risk: setup docs accidentally include MongoDB URIs, Stripe keys, webhook secrets, JWT secrets, bearer tokens, or dashboard payloads.

Mitigation: write only redacted statuses and run a secret-looking value scan before accepting the artifact.

### T-09-02: False Configuration Confidence

Risk: rows are marked configured without enough detail for a verifier to know which staging environment was checked.

Mitigation: require provider label, app root, public origin where safe, redacted environment/database label, status, evidence owner, and blocker notes.

### T-09-03: Frontend Staging Build Uses Wrong API URL

Risk: the static frontend is built before `REACT_APP_API_URL` points to the staging backend and keeps calling localhost.

Mitigation: make frontend build-time configuration a required setup row and a later runtime smoke check.

</threat_model>
