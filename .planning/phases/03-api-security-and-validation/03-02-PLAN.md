## Recommendation

Use strict schemas and DTO mappers at route boundaries, then keep controllers simple. Rejection is safer than silent persistence for unknown write fields in this phase because the spec explicitly requires that client-controlled fields such as `isAdmin` and unexpected product/coupon/address keys cannot pass through.

## Artifacts This Phase Produces

- New generic validation middleware.
- Resource validators for auth, product, cart, order, coupon, and contact.
- Updated routes/controllers to use sanitized DTOs instead of raw `req.body`, unbounded `req.query`, or unvalidated route params.
- Backend tests proving unknown fields, bad params, invalid query values, and oversized/invalid payloads fail with stable envelopes.

<threat_model>
Threats addressed: mass assignment, client-set admin privilege, malformed ObjectIds producing database error leakage, unbounded product queries, unsafe numeric coercion, and raw request bodies reaching Mongoose create/update calls. This plan does not redesign persistence models or add new ecommerce features.
</threat_model>
