## Recommendation

Treat dependency work as evidence-driven. Apply safe patch/minor upgrades first, verify with tests/build/audits, and only document a major migration if the final audit evidence proves a patch/minor path is not defensible. For frontend auth, use session storage now and document that bearer tokens are still XSS-sensitive.

## Artifacts This Phase Produces

- Updated backend and frontend package manifests/lockfiles for targeted patch/minor security remediation.
- Updated frontend auth persistence using session storage.
- Updated frontend public MapTiler config behavior without a source fallback key.
- New or updated frontend tests for auth persistence and public config behavior.
- New `03-SECURITY-RISK-REGISTER.md` if any production audit findings remain.
- Updated `docs/API.md`, `docs/CONFIGURATION.md`, and `docs/TESTING.md`.
- Final Phase 3 verification evidence suitable for the summary.

<threat_model>
Threats addressed: known vulnerable production dependencies, long-lived browser token persistence, accidental treatment of browser map keys as secrets, silent production reliance on demo config, and undocumented accepted security risk. This plan does not perform CRA-to-Vite, Express 5, Mongoose 9, React Router 7, refresh-token, or cookie-session migrations unless final audit evidence makes them unavoidable.
</threat_model>
